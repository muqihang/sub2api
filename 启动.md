本机自用最省事就是用 Docker Compose 跑（它会一起拉起 sub2api + PostgreSQL + Redis），你不用在系统里单独装数据库。我已经帮你把本机自用的配置写好了：sub2api/deploy/.env• 只允许本机访问：BIND_HOST=127.0.0.1• 端口用 18080（避免占用你本机的 8080）• 管理员账号：admin@sub2api.local / admin123456启动方式（照抄执行）bashcd /Users/muqihang/chelingxi_workspace/sub2api/deploy
docker-compose up -d
docker-compose logs -f sub2api
然后浏览器打开：http://localhost:18080就你一个人用，还需要 PostgreSQL 和 Redis 吗？• PostgreSQL：需要。它存“用户/管理员、上游账号、你生成的 sk-...、用量统计、配置”等持久数据；没它这些就没地方保存。• Redis：也建议保留（项目默认依赖它）。不只是“会话粘连”，主要用途包括：• API Key 认证的二级缓存（减少每次都查库）• 限流/并发控制（rate limiter、并发槽位等）• 仪表盘统计缓存/聚合加速• 网关调度相关的缓存（包括文档里提到的“粘性会话/调度”这类能力）大概占多少内存• 空闲时通常几百 MB～2GB 左右；实际用起来建议给 Docker 预留 2–4GB 更稳（主要是 PostgreSQL + Sub2API 自身）。如果你希望用 8080 端口，把 sub2api/deploy/.env 里的 SERVER_PORT=18080 改成 8080 即可。
