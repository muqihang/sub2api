# Formal Pool 实时状态看板运维说明

## 目标

本次改动包含两个相互独立的部分：

1. **原账号列表布局优化**
   - 账号管理页仍然是原来的管理表格。
   - 不改变筛选、编辑、诊断、上号、调度等业务逻辑。
   - 只把表格改为更紧凑、更不容易被挤压，方便一次看到更多账号。

2. **新增“号池实时看板”**
   - 在账号管理页点击 `号池实时看板` 打开全屏看板。
   - 看板是只读观察面板，不会修改账号状态。
   - 看板打开后默认每 5 秒刷新一次，关闭后停止刷新。
   - 看板只显示后端整理好的安全状态，不在前端重新判断账号状态。

## 看板状态说明

| 状态 | 含义 | 建议动作 |
| --- | --- | --- |
| 正常 | 老账号或健康账号处于可用状态 | 继续观察 |
| 预热中 | 新账号已通过门禁，正在低权重预热 | 继续预热 |
| 生产中 | 账号已进入生产状态并且证据完整 | 保持观察 |
| 限流冷却中 | 触发 429 或限额窗口，正在等待恢复 | 等待恢复，不要当坏号隔离 |
| 账号风险，需人工介入 | 出现 401、403、hold、KYC、unusual activity、risk 等风险信号 | 停止自动恢复，人工检查账号 |
| 错误 | 非限流、非人工风险的错误状态 | 打开诊断面板排查 |
| 已隔离 | 账号已被安全隔离 | 查看隔离原因和 risk event |
| 已停用 | 账号被手动停用或 inactive | 确认是否需要重新启用 |
| 不可调度 | 调度 gate 未通过，例如临时不可调度或证据 gate 阻断 | 查看 gate 原因 |
| 证据不足 | 缺少 runtime 注册或健康检查证据 | 先补 runtime/healthcheck 证据 |
| 数据不足 | 运行指标读取失败或缺失 | 检查 Redis/运行指标读取 |

## 字段解释

### 5 小时限额

看板中的 `5 小时限额` 显示：

- 已用金额
- 总限额
- 剩余额度
- 使用百分比。后端返回的是 0 到 1 的比例，看板会换算成百分比，例如 `0.25` 显示为 `25.0%`。
- 恢复或窗口结束时间

如果数据不可用，会显示 `数据不足`，不会显示成正常。

### RPM 实况

`RPM 实况` 显示当前分钟请求数和账号配置上限。

示例：

```text
2 / 3 RPM (66.7%)
```

如果账号未配置 RPM，会显示 `未配置 RPM`；如果数据源不可读，会显示 `数据不足`。

### 并发

`并发` 显示当前占用槽位和账号并发上限。

示例：

```text
1 / 1 并发 (100.0%)
```

### 会话

`会话` 显示当前活跃会话数和配置的会话上限；未配置时显示 `未配置会话`。

### 最近请求

看板优先显示账号最近使用时间或最近健康证据时间，用于判断账号是否正在被实际使用。

### 建议动作

建议动作由后端状态分类生成，前端只负责展示。常见建议包括：

- 继续观察
- 继续预热
- 等待恢复
- 人工介入
- 查看隔离原因
- 补齐证据
- 补齐数据

## 安全边界

看板是**只读**功能：

- 不发送真实 Anthropic 请求。
- 不执行 directed healthcheck。
- 不修改账号状态。
- 不改变调度策略。
- 不改变 request body 或 response body。
- 不限制 Claude Code 的 1M、tools、thinking、stream、Opus/Sonnet、max_tokens 等能力。

## 脱敏规则

看板不得展示以下高敏内容：

- setup token
- access token
- refresh token
- raw prompt
- raw body
- raw CCH
- raw telemetry
- proxy password
- 其他明显凭证或原始请求内容

运营识别需要的账号名称、邮箱可以展示；如果后端判断账号名称疑似包含 token、长密钥、代理凭证、raw 请求内容或账号/组织 UUID 等高敏内容，会退回显示为 `账号 #ID`。

## 使用流程

1. 打开后台账号管理页。
2. 点击 `号池实时看板`。
3. 先看顶部汇总卡片：
   - 正常/生产中/预热中数量
   - 限流冷却数量
   - 账号风险需人工介入数量
   - 错误/隔离数量
   - 当前总 RPM
   - 5 小时总体余量
4. 如需排查，使用筛选按钮：
   - 限流
   - 账号风险需人工介入
   - 不可调度
   - 证据不足
   - 数据不足
5. 对异常账号再打开原账号列表中的诊断面板处理。

## 故障判断

- 如果是 `限流冷却中`：通常等待窗口恢复，不要立即当坏号处理。
- 如果是 `账号风险，需人工介入`：不要自动恢复，先人工检查账号是否 hold、KYC 或异常活动。
- 如果是 `证据不足`：先补 runtime mapping、健康检查证据。
- 如果是 `数据不足`：优先检查运行指标读取，例如 Redis 或窗口费用统计。
- 如果是 `不可调度`：查看 gate 原因，确认是否临时不可调度、调度证据缺失或手动停用。

npm run test:run -- FormalPoolStatusDashboardModal formalPoolStatusDashboard
