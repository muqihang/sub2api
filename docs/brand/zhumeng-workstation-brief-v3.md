## 1. 一句话战略与三层栈

**一句话:** 让用户已经在用的顶级 Agent 工具,跑任意模型、原生级体验;用这个楔子获客,用网关和治理赚钱,用沉淀的执行轨迹(未来)训练自有模型。

**三层栈(每层喂下一层):**

```
① 接管层   让顶级 Agent 工具跑任意模型     = 获客楔子 + 协议适配护城河   【现在主战场】
                │ 产生流量
② 网关层   本地网关 + 路由 + 卖 Token + 治理 = 现金流 + 团队/企业 durable 收入 【现在主战场】
                │ 产生轨迹
③ 数据层   Agent 全链路执行轨迹 → 自有模型   = 终局护城河                  【留缝,法规清后开】
```

**Claude 的核心定性:**
- 卖 Token 是**会归零**的现金;
- 软件/治理订阅是**稳定但有上限**的现金;
- **唯一能长成"公司级资产"的是第三层**(轨迹数据 → 自有模型)。
- 但第三层有未决合规问题,**现在不能押注它,只能为它留管道**(见 §8.6)。

## 2. Claude 的核心判断(本文档的脊梁)

这一章是 v3 相对 v1/v2 最大的增量。下面每条都是**可被反驳的判断**,不是中立陈述。

### 判断 A：现在唯一该做的是"单工具做到现象级",不是"多工具市场" ✅ 已拍板

- **决议:** V0.1/V1 **死磕 Codex + Claude Code CLI 两个旗舰**,做到原生级丝滑成为圈内标志体验。长尾多工具砍到 V2+。
- **唯一例外:** 允许再加**第 1 个**工具,**前提是接入成本极低**(例如 DeepTutor 这类开源、不需要深度二进制 patch 的)。但它走"上架轻集成"档,**不占用旗舰深度接管的工程预算**。
- **判断依据(保留):** 现阶段团队小、刚完成一个工具接管,铺多工具是摊薄护城河——你真正抄不动的本事是把**一个**工具做到原生级;摊到 10 个上,每个 30 分,哪个都没壁垒,还要追 10 家上游版本。

### 判断 B：本地网关是脊柱,`app.asar` patch 是易碎的锦上添花

- **现状支撑:** `proxy/server.py` 已经是一个本地 OpenAI 兼容代理。这就是脊柱,且已存在。
- **判断:** "接管"有两种机制,稳健性天差地别:
  - **稳健核心 = 本地网关**:工具把请求指向 `127.0.0.1:<port>`,网关做协议变形 + 路由。上游改版几乎不影响。
  - **易碎外壳 = `app.asar` patch**:改 model-picker、解锁插件市场。上游**重构一次客户端就失效**。
- **推荐架构原则:** **以网关为本,patch 优雅降级。** patch 失败时,基础对话(走网关)必须照常可用,只是少了"模型选择 UI / 插件市场"这些 UX 糖。**绝不能让产品的可用性命悬于 patch。** 这才是对冲"上游风险"的正解——比"追版本快"更根本。
- 这也回答了负责人对上游风险的乐观:真实风险不是"被封模型",是"被迫永远追 patch point";解法是**把价值重心放在它们改不到的网关层**。

### 判断 C：个人侧 V0.1 不收订阅费 ✅ 已拍板

- **决议:** 个人侧 V0.1 **不设订阅**。变现靠**网关 Token 毛利**(用托管模型赚差价)+ BYOK 免费(用自己 key,赚口碑和漏斗)。
- 订阅(Pro)**留到有真 Pro 价值**(配置云同步、智能路由、用量分析、未来治理)再上,首发也应极低价 + 长免费档。
- **判断依据(保留):** 你在和**免费**的 Trae / 通义灵码 / 文心快码抢个人用户,上来收 ¥39 会直接掐死漏斗。个人不是利润中心,是漏斗和(未来)数据来源。

### 判断 D：durable 的钱在"团队治理",不在"帮个人省钱"

逐项体检第一、二层的三个价值:

| 价值 | 付费意愿 | 是否 durable |
|---|---|---|
| 省钱(顶级工具跑便宜模型) | 高(重度个人) | ❌ 会归零(token 利润趋零 + BYOK 自助化) |
| 原生体验(零配置、插件市场解锁) | 中 | 🔶 半护城河(靠持续丝滑) |
| **治理**(团队 Key 管控、成本配额、审计、合规、数据不出域) | **高(团队/企业)** | ✅ 几乎不被卷(是组织控制权) |

- **结论:** 个人侧的"省钱+原生"是**获客楔子**;团队/企业侧的**治理**才是 durable 收入。
- v1/v2 把最 durable 的那层只写了一行("Key Vault + 审计"),是个明显缺口。治理产品需要认真设计(见 §5.3),但**晚于**个人漏斗落地。

### 判断 E：双线分割现在别重投入 ✅ 已拍板

- **决议:** **只在国内注册一个主体**,做干净的国内软件产品(国产已备案模型 + BYOK)。国际版**只挂一个不做任何宣传的网页**(含 GPT/Claude/Gemini,TOS 声明不向大陆提供),不建第二个主体、不建第二个品牌、不建隔离团队。"用户自己能找到就找到。"
- **真正的纪律是一条操作规则(见 §7.3):** bottom-up 进**大陆主体企业**的单子,**一律转国产合规版**;境外模型的国际版只卖给**非大陆主体**。
- **判断依据(保留):** 对"初版刚跑通"的阶段,双主体/双品牌/物理隔离团队这套机器**过重、过早**。PMF 之前先别建两个组织;真正跨境收入起来了再建,那时它才是必要,现在是负担。

### 判断 F：数据飞轮——留缝、默认不采、法规清了再开

- 负责人:不一定推后,查完法规若合规就尽快上。
- **Claude 的判断:** 方向认同,但**别让飞轮拖慢第一、二层 ship**。
- **推荐执行顺序:**
  1. 第一、二层落地时,把 `episode_id / turn_id` 这套管道**缝留好**,但**默认只采 L0/L1(指标 + 协议形状),不采任何原文**;
  2. 法规 + 用户授权机制就绪后,**拧开关**进 L2/L3;
  3. 留缝几乎零成本,堵死了将来重构的大坑。详见现有 `docs/zhumeng-agent-training-data-flywheel-design.md`,它已把混合采集架构设计得很完整,v3 只做**节奏约束**。

## 3. 客户与价值分层

```
个人侧    价值 = 魔法(便宜 + 原生 + 零配置 + 插件市场)
          角色 = 获客漏斗 →(未来,自愿)数据贡献
          变现 = Token 毛利(订阅延后)

团队侧    价值 = 治理(Key 管控 / 成本配额 / 用量审计 / 配置下发)
          角色 = durable 收入主体("金矿")
          变现 = 座位订阅 + Token 毛利

企业/高校 价值 = 掌控 + 合规 + 数据不出域 + 私有化
          角色 = 高客单 + 标杆
          变现 = License + 部署 + SLA;数据永不入池(并以此为卖点)
```

**GTM 主线:bottom-up 多楔子 + land-and-expand。**
个人发现并爱上某个被增强的工具 → 推荐给团队 → 团队买治理 → 企业谈私有化。每个工具是一根独立楔子,插进同一个后端(网关 + 治理)。

## 4. 产品形态与现状对齐

### 4.1 产品命名

- 命名(逐梦 Agent / 逐梦 Studio / 逐梦工作台)负责人已表示**不是纠结点**,本文统一用 **逐梦 Agent** 指当前桌面产品,**逐梦 Studio** 指其 CLI 化后的完整形态。最终对外名留待品牌组。

### 4.2 模块与现状

| 模块 | 作用 | 现状 |
|---|---|---|
| 接管层(Adapters) | 让目标工具跑任意模型 | Codex **已完成**;Claude Code **进行中** |
| 网关层(本地代理) | 协议变形 + 路由 + BYOK + 托管 | `proxy/server.py` **已有**,需扩路由/多 upstream |
| 桌面前端 | 接入向导 + 状态 + 诊断 + 插件推荐 | **已有**,打磨样式 + 加插件推荐位 |
| CLI 化(逐梦 Studio) | 工作台能力暴露为 CLI,供 Agent/脚本调用 | **V0.1 预留接口,不实现** |
| 治理控制台 | 团队 Key/成本/审计/下发 | 未建,**V1 重点设计** |
| 数据管道 | episode/turn 串联 | **留缝,默认不采原文** |

### 4.3 CLI 化的战略地位(Claude 上调其优先级)

负责人把 CLI 化当"V0.1 不做、预留接口"。**Claude 认为它的战略地位被低估了:**
- CLI / headless 形态 = 让"逐梦能力"成为**任意 Agent 都能指向的本地端点**,是判断 B(网关脊柱)的自然延伸,也是**对冲上游 patch 风险**的关键(不依赖任何工具的二进制)。
- **推荐:** V0.1 实现可以不做,但**架构上要把"核心能力 = 一个可被 CLI/网关调用的本地服务"当成既定事实来设计**(现状的 proxy 已经是雏形),前端只是它的一个 GUI。不要把能力锁死在桌面 GUI 里。

## 5. 商业模式与定价

### 5.1 定价原则(Claude 版)

1. Free 必须真可用(对抗国内免费大厂);
2. **个人不靠订阅,靠 Token 毛利**(判断 C);
3. Pro 的价值锁在"软件/治理能力",不锁在"模型访问";
4. **团队座位是主利润**;
5. 价格远低于国际同行,但治理能力对标国际。

### 5.2 档位(草案,数字待市场验证)

| 档位 | 目标 | 包含 | 价格 |
|---|---|---|---|
| **Free** | 个人尝鲜 | 全部已支持工具 + BYOK 任意供应商 + 手动路由 + 基础用量看板 | ¥0 |
| **Token 托管** | 没 key / 嫌麻烦的个人 | 用逐梦托管模型,按量计费 | Token 毛利(国际版才含境外模型) |
| **Pro(延后)** | 个人重度 | Free + 智能路由 + 配置云同步 + 完整用量分析 | 上线时定,低价 + 长免费档 |
| **Team** | 5-20 人团队 | Pro + Key Vault(RBAC) + 成本配额 + 用量审计 + 配置下发 + 管理台 | 座位/月(主利润) |
| **Enterprise** | 企业/高校 | 私有化 + 合规 + 数据不出域 + SSO + 定制 + SLA | 商务洽谈 |

### 5.3 团队治理产品(v1/v2 的最大缺口,这里补上)

团队买你,**具体图什么**——这是 V1 必须回答的:

- **Key 管控:** 团队统一管理供应商 key / 逐梦托管额度,成员不接触明文 key(Key Vault + RBAC)。
- **成本配额:** 按成员/项目设 token 预算与告警,防止失控。
- **用量审计:** 谁、用了什么模型、花了多少、在哪个工具里——管理员可见。
- **配置下发:** 把"用哪些模型、哪些工具、什么路由策略、什么安全策略"统一下发到成员的逐梦 Agent。
- **合规/不出域:** 私有化部署形态下,数据流不经过任何外部(含逐梦)服务器,并提供审计证据。

> ⚠️ 治理是 durable 收入的真正来源,但**晚于个人漏斗**。V0.1 不做,V1 重点设计。

## 6. 护城河

按"抄走难度"排序:

1. **协议适配深度(旗舰工具原生级体验)** —— 现在的主战场,赢家通吃特征。**注意:不是"智能路由"。** 智能路由是 if-else 规则引擎,工程含量低,别拿它当护城河对外吹。
2. **团队治理粘性** —— 一旦团队把 key/成本/审计/配置标准化在逐梦上,迁移成本极高。
3. **(未来)Agent 轨迹数据 → 自有模型** —— 终局,留缝延后。
4. **覆盖广度的"维护跑步机"** —— 多模型 × 旗舰工具持续可用,是别人要持续投入才能追的成本壁垒(但前提是判断 A:先做深,别先做广)。

## 7. 合规姿态(精简版)

### 7.1 国内线:做一个干净的软件

- 桌面软件 + 国产已备案模型 + BYOK,**法律定性是"软件供应商"**,站得住。
- 国内渠道**只讲软件 + 国产模型 + BYOK + 治理**,不主推境外模型。

### 7.2 国际线:低调单独跑

- 国际版含 GPT/Claude/Gemini,**TOS 声明不向中国大陆提供服务**;主体/域名/支付与国内线分开(但**现在不必建重型双主体机器**,见判断 E)。
- 现实认知:大陆个人会翻墙自用——这部分**看不见、不主动服务、允许存在**,但**绝不在国内渠道引流**。

### 7.3 关键操作纪律(比建双主体更重要)

> **bottom-up 进大陆主体企业的单子,一律转国产合规版(只给已备案模型);境外模型的国际版,只卖给非大陆主体。**

这是把"个人翻墙自用"和"向大陆企业提供未备案境外模型服务"两件事**切开**的那把刀。GTM 越是 bottom-up,这条纪律越要写死,否则 bottom-up 会亲手捅穿合规墙。

> ⚠️ 本章是结构性提示,非法律意见。最终以负责人正在核实的具体法规为准(生成式 AI 服务备案、深度合成算法备案等)。

## 8. 路线图与"现在不做什么"

好战略首先是**敢说不**。

### V0.1（最近期,在手进度上收口）
**做:**
1. Codex 接管打磨稳定(已完成主体);
2. Claude Code CLI 接管跑通;
3. 桌面前端样式打磨 + 插件推荐位;
4. 本地网关:BYOK + 逐梦托管 + 手动选模型;
5. 数据管道**留缝**(episode/turn id),默认只采 L0/L1;
6. 关键架构原则落地:**网关为本,patch 优雅降级**。

**不做:**
- ❌ 个人订阅收费;
- ❌ 长尾多工具(DeepTutor/Openclaw/Hermes…);
- ❌ 团队治理控制台;
- ❌ 智能路由(自动);
- ❌ CLI 化实现(只预留接口);
- ❌ 重型双主体/双品牌机器;
- ❌ 数据飞轮原文采集。

**V0.1 的唯一成功标准(不是功能清单,是一个数字):**
> 把 Codex 接管推给 ~100 个**陌生**用户,看**有多少人会回来跑第二个任务**。
>
> 这个留存数字若起不来,任何后续战略都先暂停——说明"在 Codex 里跑 DeepSeek"这个核心假设还没被市场验证。功能做得再全,都不如这个数字真实。

### V1（个人漏斗成立后）
- 智能路由、配置云同步、用量分析 → 上 Pro;
- **团队治理控制台**(§5.3)→ 开始赚 durable 的钱;
- CLI 化(逐梦 Studio)实现;
- 企业私有化方案文档 + 首批标杆。

### V2+（漏斗 + 治理都验证后）
- 长尾工具上架(轻集成),做"多楔子市场";
- 数据飞轮(法规 + 授权就绪后开 L2/L3);
- 视行业格局,考虑开源 `workstation-tools` / `gateway-lite`(协议适配层始终闭源)。

## 9. 已拍板决策(2026-06-15)

下列五条原为"分歧待拍板",现已全部按 Claude 推荐采纳,作为后续执行的**约束条件**:

1. **焦点 = ✅ 死磕 Codex + Claude Code CLI 两个旗舰**;长尾砍到 V2+;唯一例外是允许加 1 个"极低成本上架"的工具(如 DeepTutor),且不占旗舰工程预算。
2. **个人定价 = ✅ V0.1 不收订阅**,纯靠 Token 毛利;Pro 延后到有真价值再上。
3. **合规结构 = ✅ 只注册一个国内主体**,做干净国内软件;国际版只挂不宣传的网页;靠"bottom-up 进大陆企业转国产合规版"这条纪律守边界,不建双主体双品牌。
4. **CLI 化 = ✅ 上调为脊柱级架构事实**:核心能力 = 一个可被 CLI/网关调用的本地服务,GUI 只是它的壳;V0.1 不实现完整 CLI,但架构按此设计、预留接口。
5. **数据飞轮 = ✅ 留缝 + 默认不采原文 + 法规清后再开**;不得拖慢第一、二层 ship。

## 11. 执行纪律(v3 复盘新增,最该被记住的部分)

战略已经够用,真正的瓶颈在执行。以下四条是 Claude 在复盘整个项目(代码 + 四份文档)后给出的、比战略更要紧的提醒:

### 11.1 规划深度只允许领先现实一个版本
现象:已接管**一个**工具,却写了 2300 行飞轮方案 + 1100 行桌面设计 + 三版 brief。这是"想得太远以至于文档变成已在前进的错觉"。
**纪律:** V1 ship 之前不写飞轮 RL 方案;每份文档的细节深度,不得超过当前真实进度一个版本。

### 11.2 验证优先于建设
四份文档里没有一个真实用户、一块钱收入。**当前瓶颈不是战略,是验证与分发。**
**纪律:** 在做下一个工具(Claude Code)之前,先把已有的 Codex 接管丢给真实用户、拿到 §8 那个留存数字。建设服从验证。

### 11.3 信任门槛 = 真正的 GTM 难题,开源是它的硬解锁
目标用户(注重隐私的开发者)凭什么让一个**没听过的闭源二进制**去 patch 他的 IDE、经手他的 key?这是纯技术跨不过的门槛。
**纪律:** **经手用户 key / 流量的那一层(本地网关 shim)应尽早开源**——这不是"信任红利"的锦上添花,是分发的硬解锁。协议适配的深度逻辑仍闭源,但"碰 key 的壳"必须可审计。这一条会**提前**原 v2 开源时间表里 `gateway-lite` 的优先级。

### 11.4 严谨要分级,不要均匀用力
现有文档对所有事都用 100 分严谨。**100 分平摊 = 慢。**
**纪律:** 100 分留给——协议适配、key 安全、不搞坏用户配置;60 分给——长尾工具、远期飞轮、UI 像素。把工程鞋底磨在刀刃上。

> **v3 一句话总结:** 以**本地网关为脊柱**、以**单工具现象级为楔子**,个人侧用魔法和 token 毛利做漏斗、团队企业侧用治理赚 durable 的钱;合规靠"干净国内软件 + 低调国际 + 转化纪律"而非重型双主体;数据飞轮留缝延后。**先把一件事做到 100 分,再谈第二件。**
