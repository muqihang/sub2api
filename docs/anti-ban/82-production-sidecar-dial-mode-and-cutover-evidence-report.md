# Plan82 Production Sidecar Dial Mode and Cutover Evidence Report

## Final decision

**PASS_PRODUCTION_SIDECAR_FULL_CUTOVER**

Plan82 implemented production dial mode for the Go/uTLS sidecar, validated it without test dial override, and cut the production `18080` egress path to sidecar-enabled CC Gateway. No rollback was required.

## Code change summary

- Added explicit sidecar dial mode:
  - `EGRESS_TLS_SIDECAR_DIAL_MODE=production|test`
  - default: `production`
- Production mode:
  - does not require test dial override;
  - rejects test dial override if present;
  - installs no dial override map;
  - keeps `AllowTestDialOverride=false`;
  - keeps allowed target host `api.anthropic.com` and port `443` only.
- Test mode:
  - preserves previous loopback-only `EGRESS_TLS_SIDECAR_TEST_DIAL_OVERRIDE_API_ANTHROPIC` behavior for local/mock smoke.
- Safety tests added/updated:
  - production mode without override PASS;
  - production mode with override fail-closed;
  - test mode missing override fail-closed;
  - unknown dial mode fail-closed;
  - non-allowlisted target host fail-closed;
  - request-controlled `dial_override`, `server_name`, `proxy_url`, `sni` remain rejected.
