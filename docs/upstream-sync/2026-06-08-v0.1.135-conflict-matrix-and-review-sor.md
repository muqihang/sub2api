## 1. Decision principles

### 1.1 Why not merge version-by-version

The upstream range contains hundreds of commits across schema, gateway routing, billing, quota, payments, frontend, and compatibility paths. Merging 12 release tags one by one would create repeated conflict churn in the same hot files (`gateway_service.go`, `openai_gateway_service.go`, DTOs, settings, pricing JSON, frontend account/settings pages). The safer approach is one target merge into an isolated worktree, then resolve by system boundary and checkpoint.

### 1.2 Two review lenses

Every conflict must be reviewed from both perspectives:

1. **Upstream benefit lens:** keep real `v0.1.135` improvements in product capability, correctness, accounting, billing, security, and frontend UX.
2. **Local safety/capability lens:** do not regress the local fork's Claude formal-pool safety, CC Gateway routing, Claude Code parity, session/account budget observability, Codex Gateway, DeepSeek/AGNES/Computer Use fidelity, and sensitive scan rules.

### 1.3 Hard invariants that cannot be broken

- Do not weaken Claude formal-pool onboarding hard gates: new accounts default unschedulable, runtime registration before production, directed healthcheck before production, warming before production, automatic quarantine for identity/proxy/auth/risk failures.
- Do not reintroduce direct managed-Claude fallback that bypasses CC Gateway.
- Do not leak raw token, raw prompt, raw body, raw telemetry, raw CCH, email, account/organization UUID, or proxy credentials in admin DTOs, ops logs, captures, or docs.
- Do not remove or downgrade safe-deliverable sensitive scanning.
- Do not remove session/account/user/pool budget ledgers or observe-only decision outputs.
- Do not weaken normal/aggressive pool protection: aggressive can be requested but must not apply to new/warming accounts.
- Do not break Claude Code compat adapter behavior: Anthropic-only ingress, fail-closed protocol checks, shape normalization, ToolSearch/deferred/capability-backed logic, native marker preservation, and official Claude Code traffic staying native.
- Do not regress Codex Gateway high-fidelity behavior: DeepSeek/AGNES/Claude Computer Use visible-text budgets, reasoning preservation, tool continuation, Codex Desktop capture and route behavior.
- Do not accept generated conflict output without source-level verification and test coverage.

### 3.4 Local-safety reviewer addendum

Read-only local-safety review identified these additional non-negotiable protections:

- **Formal-pool lifecycle:** `schedulable=true` may only appear in the allowed lifecycle stages; new/unsafe Claude accounts must not be made routable by upstream account creation defaults.
- **CC Gateway evidence chain:** keep `cc_gateway_seen`, runtime registration state, account/proxy safe refs, egress bucket, raw-capture presence, safe error buckets, and risk event references. Missing evidence, proxy mismatch, fallback, or risk text must fail closed or quarantine.
- **Native Claude Code parity:** UA/originator, `metadata.user_id`, count-tokens, probe handling, billing header/CCH/tool-rewrite/API-key passthrough cannot be simplified to single-factor detection.
- **Session budget:** `SessionBudgetObserveSink` and `SUB2API_SESSION_BUDGET_EXPORT_PATH` must survive the merge and remain observe-only unless explicitly changed.
- **Codex/AGNES/DeepSeek/Computer Use:** preserve provider groups, model registry, pricing, frontend whitelist, `tool_search_call/output`, Computer Use visible-text fidelity, subagent/skills/tool continuation, native image/thinking behavior.
- **Logging/DTO redaction:** ops error, logger, DTO, frontend scrubbers must continue returning safe refs/buckets only.

Reviewer warning: the following decisions require manual fusion and should not be solved by choosing a side mechanically: Claude Code native marker shape, `metadata.user_id`/session-id stability vs correlation risk, `buildUpstreamRequest` CCH/billing-header/passthrough behavior, OpenAI WS state/sanitize/TLS/egress behavior, OpenAIGateway constructor dependencies, ent generated files, and pricing JSON union.

## 7. Conflict resolution heuristics

1. **Generated files:** source first, regenerate second, inspect generated output third.
2. **Config structs/defaults:** additive union unless two defaults are semantically incompatible. If incompatible, prefer safer default and document why.
3. **Admin DTOs:** never return raw credentials. If upstream adds fields, map them through local redaction helpers.
4. **Scheduler/account state:** never make formal-pool Claude accounts schedulable unless lifecycle stage allows it.
5. **Gateway body handling:** prefer upstream `RequestBodyRef`/memory fixes, but keep local shape normalization, compat audit, and session-budget observation hooks.
6. **Anthropic/Claude:** local fail-closed CC Gateway path dominates; upstream direct passthrough improvements are allowed for unmanaged/API-key paths only when tests prove no managed-account bypass.
7. **OpenAI/Codex:** upstream quota/Responses/WS fixes are desired; local DeepSeek/AGNES/Computer Use high-fidelity paths must remain test-covered.
8. **Frontend:** if both sides add controls, preserve both; consolidate into dropdowns only if labels/routes/tests show no lost action.
